# 文章深度分级质量分析后台

一款**纯前端**的内容质量数据分析工具，面向质量管理团队，实现本地质量数据上传、可视化大盘展示、多维度问题归因分析与明细查询，**数据全程存储于浏览器本地，不与任何服务器交互**。

## 本地运行步骤

### 环境要求
- Node.js >= 18
- pnpm >= 8（或 npm/yarn）

### 安装与启动

```bash
# 克隆或进入项目目录
cd article-quality-dashboard

# 安装依赖
pnpm install

# 启动开发服务器（默认 http://localhost:5173）
pnpm run dev

# 生产构建
pnpm run build

# 预览生产构建
pnpm run preview
```

## 功能说明

### 模块1：数据上传管理（`/upload`）
- **拖拽/点击上传** `.xlsx` / `.csv` 文件，单文件最大 100MB，最多 10 万行
- **两种上传模式**：追加数据 / 覆盖全部数据（覆盖前二次确认）
- **表头自动校验**：解析后自动比对字段完整性，显示前 10 行预览
- **错误明细展示**：逐行校验日期格式、数字范围、枚举值，标注错误行号与原因
- **上传历史管理**：记录所有上传批次，支持删除单批次数据
- **一键清空**：二次确认后清空所有本地数据

### 模块2：质量数据总览看板（`/dashboard`）
- **6 大核心指标卡**：准确率、文章分级准确率、深度分级准确率、错审率、漏审率、归类错误率
- **环比趋势**：每个指标展示与上期对比差值和颜色标识（绿色=变好，红色=变差）
- **准确率趋势折线图**：支持按日/周/月切换时间粒度
- **质检结论分布饼图**：直观显示正确/错审/漏审/归类错误比例
- **全局筛选联动**：时间范围、质检结论、审核结果、质检结果
- **数据导出**：支持导出看板明细、趋势数据、分布数据为 Excel

### 模块3：审核员维度分析（`/auditor`）
- **按审核员汇总统计**：展示每位审核员的质检量、错审量、漏审量、归类错误量、准确率
- **多维排序**：支持按任意列正序/倒序排序
- **准确率柱状图**：可视化对比各审核员准确率，低于阈值自动标红
- **数据导出**：导出审核员分析 Excel

### 模块4：标签归因分析（`/label`）
- **分析归类错误数据**：聚焦 `qc_conclusion=归类错误` 的样本
- **标签 Diff 计算**：对比审核标签（JSON解析）与质检标签（0/1二值）差异
  - 误判：审核=命中，质检=未命中（多给了标签）
  - 漏判：审核=未命中，质检=命中（少给了标签）
- **水平条形图**：直观展示各标签误判率与漏判率分布
- **标签1/标签2 全覆盖**：11个标签的完整归因分析

### 模块5：数据明细查询（`/detail`）
- **多条件组合筛选**：时间范围、质检结论、审核结果、质检结果、审核员、文章标题模糊搜索、链接搜索、audit_id精准匹配
- **虚拟分页**：前端分页展示，支持 20/50/100/200 条/页
- **列自定义**：通过"列设置"控制显示/隐藏的字段列
- **多选导出**：勾选单条或多条记录导出 Excel
- **全量导出**：一键导出当前筛选结果全量数据

## 数据格式说明

### 支持的文件格式
- `.xlsx`（Excel，推荐）
- `.csv`（逗号分隔值）

### 表头规范
文件第1-2行为合并表头（部分字段有父级分组），系统自动合并处理。

### 关键字段

| 字段 | 说明 |
|------|------|
| `qc_conclusion` | 质检结论，枚举值：`正确/错审/漏审/归类错误`，**所有指标计算的核心基准** |
| `audit_label_1` / `audit_label_2` | 审核标签，JSON格式，如 `{"list":["\"\"","表达流畅"]}` |
| `质检标签 1 子列 - XXX` | 质检标签，0/1 二值（求知价值为 0/2） |
| `是否申诉` | 枚举：`是/否` |

## 注意事项

1. **数据安全**：所有数据存储于浏览器 IndexedDB，清除浏览器数据后本地数据将丢失，请定期备份原始文件
2. **大文件性能**：10万行数据解析约需 5-15 秒，解析期间请勿关闭页面
3. **浏览器兼容性**：推荐使用 Chrome 100+ 或 Edge 100+
4. **时间范围**：默认展示近7天数据，若无数据请调整时间范围或先上传数据
5. **Excel表头**：上传文件的表头必须与系统约定字段完全一致（含空格），否则数据将无法正确解析
